Data Flow · 数据流转

1
ESP32 唤醒，通过 WiFi 发送 HTTP GET 到后端，携带电压值和 MAC 地址
2
后端从 SQLite 加载设备配置，根据刷新策略选择内容模式
3
检查内存缓存，命中则直接返回；未命中则并行预生成所有模式
4
并行获取环境上下文：日期/农历/节气 + Open-Meteo 天气 + 节假日信息
5
调用 LLM API（DeepSeek / 百炼 / Kimi）生成结构化内容
6
Pillow 渲染 400×300 1-bit 图像，转换为 BMP 二进制流返回
7
ESP32 接收 BMP 数据，通过 SPI 写入电子墨水屏显示
8
显示完成后进入 Deep Sleep，等待下次定时唤醒（10 分钟 ~ 24 小时）
